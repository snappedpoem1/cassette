# Cassette UI Flair Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Transform Cassette's UI into a living, music-reactive visual system — cinematic depth, analog grain texture, and club energy that morphs with every track change.

**Architecture:** CSS variable expansion from the existing `--mood-accent-rgb` per-track sampler in `+layout.svelte`. New variables (`--mood-glow`, `--mood-border`, `--mood-fill-a`, `--mood-fill-b`) derived from mood RGB flow to every surface. Animated backdrop blobs, SVG grain overlay, and a new `CassetteLogo.svelte` component are the only structural additions. All animations respect `ui_visualizer_low_motion`.

**Tech Stack:** SvelteKit, CSS custom properties, inline SVG, Svelte `{#key}` blocks for cross-fade, CSS `@keyframes`, `backdrop-filter`
